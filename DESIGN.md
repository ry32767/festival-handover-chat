# DESIGN — 文化祭引継ぎチャット

> UI判断の根拠を引き継ぐファイル。実際の値は`src/styles/tokens.css`を正とする。

## 題材（1文）

文化祭スタッフが、準備の途中でも迷わず引継ぎ資料へ到達し、根拠を確認して次の行動を決めるための実務画面。

## トークン

| 種別 | 名前 | 値 | 根拠 |
|---|---|---|---|
| color | `ink` | `#17211D` | 長文でも読みやすい、わずかに緑を含む濃色 |
| color | `canvas` | `#F3F6F4` | 校内の紙資料を想起させつつ、クリーム調を避けた背景 |
| color | `surface` | `#FFFFFF` | 入力面と本文の明確な作業領域 |
| color | `signal` | `#B42335` | 重要な行動だけに使う文化祭らしい赤 |
| color | `support` | `#197066` | 成功・補助情報に使う赤と区別可能な緑 |
| color | `rule` | `#C7D1CC` | 資料の罫線を思わせる境界色 |
| type (display) | `Yu Mincho` | 600、見出しのみ | 引継ぎ文書の題字らしさを抑制して使う |
| type (body) | `BIZ UDPGothic` | 400/700 | 日本語の実務文を画面で読みやすくする |
| space | 4px基調 | 4/8/12/16/24/32/48 | 密度を保ちながらタップ領域を確保する |
| radius | 4px / 6px | 小さく固定 | 運用ツールとして輪郭を明確にする |
| motion | 140ms ease-out | reduced-motion尊重 | 状態変化だけを短く知らせる |

## シグネチャ

見出し左端の太い赤罫線を「引継ぎ資料の重要マーカー」として使う。装飾はここへ集中し、画面本体は静かな罫線と余白で構成する。

## 避けると決めた初期値

- 紫のグラデーションやガラス表現は使わない。学校運営の実務画面に不要なため。
- クリーム＋テラコッタに寄せない。紙らしさより検索・確認の明瞭さを優先するため。
- カードを入れ子にしない。情報の階層は見出し、罫線、余白で示す。
